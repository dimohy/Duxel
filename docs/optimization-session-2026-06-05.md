# Optimization Session — 2026-06-05

## Scope

Continuation work toward making Duxel's Vulkan backend a faster, more structurally complete 2D renderer by reducing CPU-side upload overhead in static geometry and texture/cache creation paths.

Current focus:

- Preserve the existing static/dynamic draw-list split.
- Reduce per-resource transfer submissions by using upload batching safely.
- Remove avoidable heap allocation and extra memory copies from static vertex buffer creation.
- Split device/vendor capability policy from general renderer orchestration.
- Move static geometry content identity toward Core-owned cheap stamps instead of renderer-side scans.
- Move static layer opacity out of CPU-side alpha-rewritten geometry and into Vulkan shader/push-constant state.
- Reduce avoidable descriptor lookup/bind work for primitive commands that use the app white texture.
- Keep the renderer contract strict: unsupported paths fail explicitly instead of falling back.
